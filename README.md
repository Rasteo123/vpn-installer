# VPN Installer

Поднять VPN на сервере и настроить под него роутер — это обычно вечер возни в SSH: конфиги AmneziaWG, sing-box, nginx, потом uci на роутере, маршруты, файрвол. Эта утилита делает то же самое за тебя. Вводишь адрес сервера, ключ или пароль, пароль от роутера — и она раскатывает весь стек на обе стороны.

Под капотом — связка, которая реально крутится у меня дома: **AmneziaWG** как основной туннель, **NaiveProxy** как запасной, и автопереключение между ними, если один отвалится. Российский трафик при этом идёт мимо туннеля, напрямую — чтобы банки и госуслуги не упирались в «иностранный» IP.

## Что ставится

**На сервер (Ubuntu):**
- AmneziaWG — обфусцированный WireGuard на udp/443.
- NaiveProxy через sing-box — притворяется обычным HTTPS, требует домен.
- nginx — держит сертификат Let's Encrypt и отдаёт камуфляжный сайт на 443.

**На роутер (OpenWrt):**
- Клиент AmneziaWG (основной) и клиент NaiveProxy (резерв).
- Демон failover: проверяет оба интерфейса и переводит дефолтный маршрут на живой.
- PBR — российские домены и подсети уходят напрямую через WAN, остальное в туннель.

VLESS-Reality в коде нет, и это сознательно: на роутере он не использовался, а держать на сервере протокол, к которому никто не подключается, смысла мало.

## Честно про статус

Серверная половина **проверена вживую** — я поднимал весь стек с нуля на чистой Ubuntu 24.04: туннель строится, NAT и форвардинг на месте, камуфляжный сайт отдаёт 200. Работает.

Роутерная половина **написана и покрыта юнит-тестами, но на живом роутере ещё не обкатана**. Причина бытовая: роутер у меня один и рабочий, а прогон установщика временно подменит текущую конфигурацию. Делать это надо с бэкапом и под присмотром. В код это заложено — снимок `uci` перед любым изменением и автооткат при сбое, — но «в бою» я пока не гонял. Так что если возьмёшься первым, держи открытой SSH-сессию к роутеру и сделай полный бэкап OpenWrt заранее.

## Установка

Готовые сборки лежат в [Releases](../../releases): `.exe` (+ portable) для Windows, `.dmg`/`.zip` для macOS, `.AppImage`/`.deb` для Linux.

Они **не подписаны** — Windows и macOS будут ругаться на «неизвестного разработчика». Это нормально для опенсорса без сертификата за сотню долларов в год. На маке: правый клик по приложению → «Открыть». На винде: «Подробнее» → «Выполнить в любом случае».

## Что понадобится

- Сервер на **Ubuntu** (гонял на 22.04 и 24.04), root по SSH.
- Роутер на **OpenWrt** или совместимой прошивке. У меня — Routerich AX3000 на OpenWrt 24.10.
- Для NaiveProxy — **домен**, A-запись которого смотрит на IP сервера. Бесплатный поддомен можно завести на [dynu.com](https://www.dynu.com). Без домена Naive просто не поставится, всё остальное при этом работает.

Деталь про сертификаты, на которой легко споткнуться: Let's Encrypt выдаёт не больше 50 сертификатов в неделю на один зарегистрированный домен. На общих DDNS вроде xubi.org этот лимит общий на всех пользователей, так что упереться в него можно вообще не по своей вине. Лучше взять домен на базе поспокойнее.

## Собрать из исходников

```bash
npm install
npm run dev          # запуск в режиме разработки
npm run dist:win     # сборка под Windows x64 (NSIS + portable)
npm run dist:mac     # под macOS (dmg + zip)
npm run dist:linux   # под Linux (AppImage + deb)
```

Одна засада на маке: если в пути к проекту есть **пробел** (как в «VPN installer»), пересборка нативного модуля `cpu-features` падает. На работу это не влияет — ssh2 спокойно обходится без него, — но для чистой сборки лучше держать проект в пути без пробелов. В CI этой проблемы нет, там код лежит в нормальном пути.

Все три платформы собираются сами через GitHub Actions: на пуш в `main` — артефакты в разделе Actions, на пуш тега `v*` — готовые файлы прикладываются к релизу. Конфиг — в `.github/workflows/build.yml`.

## Как это устроено внутри

Не «скрипт, который шпарит команды по SSH и надеется на лучшее». Внутри оркестратор: установка разбита на шаги, у каждого свои фазы — проверка перед запуском, выполнение, проверка результата, откат при сбое. На роутере сперва прогоняются все проверки и только потом начинаются изменения, а при любой ошибке конфиг восстанавливается из снятого бэкапа.

Конфиги не пишутся по памяти. Они сняты с живого рабочего стека (`npm run snapshot`) и воспроизводятся шаблонами — а тесты сверяют вывод шаблона с эталоном. Тесты гоняются через `npm test` (встроенный `node:test`, без лишних зависимостей). Часть из них сверяется с тем самым снимком конфигов; снимок привязан к конкретным устройствам и в репозиторий не входит, поэтому без локального снимка эти тесты просто пропустятся.

## Безопасность

- **Проверка ключа сервера (TOFU).** При первом подключении отпечаток SSH-ключа сервера запоминается в `~/.vpn-installer/known_hosts.json`, дальше сверяется. Если ключ вдруг сменился (возможный MITM — или сервер честно переустановили), подключение прерывается с явным сообщением; чтобы доверять новому ключу, удали строку хоста из этого файла. Файл свой на каждой машине — подключение с другого компьютера просто запомнит ключ заново, ничего не сломается.
- **Секреты не светятся.** Ключи AmneziaWG и PSK генерируются на сервере так, что не попадают в аргументы команд (не видны в `ps`). Конфиги с приватным ключом и паролями пишутся сразу с правами `600`. Приватный ключ клиента и PSK не возвращаются в интерфейс. Пароль NaiveProxy — случайные 144 бита.
- **Адрес сервера, домен и порт валидируются** перед тем, как попасть в команды на стороне сервера/роутера, — опечатку с лишним символом утилита отвергнет, а не выполнит.
- **Firewall настраивается сам.** Если на сервере активен `ufw`, нужные порты (udp/443 для AWG, 80/443/2053 tcp для Naive) открываются автоматически, а при откате — закрываются. Существующий `nginx.conf` перед перезаписью сохраняется и возвращается на место при сбое.
- **Параметры обфускации AmneziaWG случайны для каждой установки** — у разных установок нет общего «отпечатка» для DPI.

На роутере недостающие утилиты (`curl`, `jq`, `ca-bundle`) ставятся автоматически — на чистом OpenWrt их нет, а проверкам и failover они нужны. Если настройка роутера срывается и автооткат из бэкапа не удаётся, утилита прямо показывает команду ручного восстановления (`uci import … < /root/vpn-installer-backup-latest.*`).

## Дисклеймер

Это инструмент под мой сценарий, а не коммерческий продукт. Гарантий никаких, всё на твой риск — особенно роутерная часть, которая трогает сетевые настройки боевого устройства. Бэкап и откат в неё встроены, но первый прогон всё равно делай аккуратно.
